86 The `recursive` keyword controls whether deeper levels of nested macros are also expanded.

112 end

114 """
115 @macroexpand
116
117 Return equivalent expression with all macros removed (expanded).
118
119 There are differences between `@macroexpand` and [`macroexpand`](@ref).
120
121 * While [`macroexpand`](@ref) takes a keyword argument `recursive`, `@macroexpand`
122 is always recursive. For a non recursive macro version, see [`@macroexpand1`](@ref).
123
124 * While [`macroexpand`](@ref) has an explicit `module` argument, `@macroexpand` always
125 expands with respect to the module in which it is called.
126 This is best seen in the following example:
127 ```julia-repl
128 julia> module M
129 macro m()
130 1
131 end
132 function f()
133 (@macroexpand(@m),
134 macroexpand(M, :(@m)),
135 macroexpand(Main, :(@m))
136 )
137 end
138 end
139 M
140
141 julia> macro m()
142 2
143 end
144 @m (macro with 1 method)
145
146 julia> M.f()
147 (1, 1, 2)
148 ```
149 With `@macroexpand` the expression expands where `@macroexpand` appears in the code (module `M` in the example).
150 With `macroexpand` the expression expands in the module given as the first argument.
151 """
152 macro macroexpand(code)
153 return :(macroexpand($__module__, $(QuoteNode(code)), recursive=true))
154 end
